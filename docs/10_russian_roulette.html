Russian roulette

10_russian_roulette.cpp — terminating an infinite sum after a random, finite number of steps without introducing any bias. The trick that lets a path tracer stop bouncing.

A light path can bounce forever: each bounce multiplies a running throughput by the surface albedo ($<1$) and adds any emission, so the radiance along a path is an infinite sum. We can’t trace infinitely many bounces. Just stopping after a fixed number throws away every later bounce — that’s bias, visible as energy loss and too-dark interreflections. Russian roulette stops early too, but compensates so the answer stays correct on average.

One-sentence summary. At each step flip a biased coin — continue with probability $q$, otherwise stop — and whenever you continue, divide the running weight by $q$; that division exactly cancels the chance of having been killed, so the estimator stays unbiased while the work per path becomes finite (expected length $1/(1-q)$). You trade a little variance for a guaranteed-finite, terminating computation.

1. The infinite sum

Model a path with constant per-bounce albedo $\rho$ and constant emission $e$ at every vertex. The radiance is a geometric series — emission at the first vertex, plus $\rho$ times the emission seen after one bounce, plus $\rho^2$ after two, and so on:

$$S=e\,(1+\rho+\rho^2+\rho^3+\cdots)=\frac{e}{1-\rho}.$$

With $e=1$ and $\rho=0.8$ the true answer is $1/(1-0.8)=5$. Every term is positive and they only shrink geometrically, so the deep bounces still matter — and dropping them all is exactly what makes naive truncation too dark.

2. Fixed truncation is biased (too dark)

Stop after $K$ bounces and you sum only the first $K+1$ terms, missing $S\,\rho^{K+1}$ of the answer — always an underestimate. Slide $K$: the cumulative estimate climbs toward $S$ but never reaches it, and the missing tail (grey bars) is the energy you discarded.

3. The roulette mechanic

Instead of a fixed cutoff, after each vertex flip a coin: continue with probability $q$, else stop. On every continuation, divide the running weight by $q$. With the throughput-matched choice $q=\rho$ the weight stays exactly $1$ (the $/q$ cancels the $\times\rho$), so every surviving vertex contributes exactly $e$ and the path length is the only random thing. 4. Why it stays unbiased

Track term $k$ of the sum. It is reached only if the path survived $k$ coin flips, which happens with probability $q^k$; and when reached, the weight has been divided by $q$ that many times, so it is scaled by $1/q^k$. The expected contribution of term $k$ is therefore

$$\underbrace{q^k}_{\text{reach it}}\times\underbrace{\frac{1}{q^k}}_{\text{weight}}\times(\text{term }k)=(\text{term }k),$$

so the expectation of the roulette estimator equals the full infinite sum — for any $q\in(0,1)$. Meanwhile the expected number of steps is itself a geometric series,

$$\mathbb E[\text{steps}]=1+q+q^2+\cdots=\frac{1}{1-q},$$

which is finite. The randomized stopping is what converts an infinite computation into a finite one without tilting the average.

5. The payoff: unbiased at any $q$

Run many paths and histogram their estimates. Wherever you set $q$, the mean sits on $S=5$ — roulette is unbiased — but the spread changes: a smaller $q$ kills paths sooner (less work) at the cost of higher variance. Slide $q$ and watch the centre hold while the width breathes.

6. Choosing $q$: the variance–cost tradeoff

If roulette is unbiased for any $q$, why not pick a tiny $q$ and stop almost immediately? Because the compensating $1/q$ blows up the surviving weight, and the variance with it. The per-path variance for this model is

$$\operatorname{Var}=\frac{e^2+2e\rho S}{\,1-\rho^2/q\,}-S^2 ,$$

which diverges as $q\to\rho^2$ from above — set $q\le\rho^2$ and the variance is infinite (fireflies). Push $q\to1$ and the variance vanishes, but the path never terminates ($\text{cost}=1/(1-q)\to\infty$). The standard choice $q=\rho$ (set from the throughput) keeps the weight exactly $1$ — bounded, no spikes — which is the practical sweet spot.

7. This is the renderer’s path termination

Rendering link. The ray-gen loop applies exactly this. After accumulating a bounce it calls RussianRouletteTerminate, defined in Utils.hlsl: it sets the survival probability to the max channel of the running throughput, clamp(max(throughput.rgb), 0.05, 0.95), terminates if a uniform draw exceeds it, and otherwise does throughput *= 1/q — the same compensation. The $q=$throughput rule is our $q=\rho$; the $0.05$ floor prevents the $1/q$ firefly blowup (our $q>\rho^2$ rule), the $0.95$ ceiling keeps paths finite, and RR is skipped for the first couple of bounces so the bright early path stays low-variance.

8. The code

The complete sample on GitHub: 10_russian_roulette.cpp.

const double e = 1.0, rho = 0.8, q = rho;       // continuation prob = albedo
for (long long p = 0; p < kPaths; ++p) {
    double s = 0.0, weight = 1.0;
    for (;;) {
        s += e * weight;                        // contribution of this vertex
        if (rng.NextFloat() >= q) break;        // roulette: terminate
        weight /= q;                            // survived: compensate ...
        weight *= rho;                          // ... then apply the bounce albedo
    }
    /* Welford over s → mean (≈5, unbiased) and variance */
}
// avg bounces/path ≈ 1/(1-q) = 5

Because $q=\rho$, the weight /= q; weight *= rho; pair leaves the weight at $1$, so the loop reduces to “count $e$ per surviving bounce.” The mean lands on $5$ (unbiased) where fixed truncation always fell short.

9. Takeaways

  • Truncation is biased. Cutting the path at a fixed depth discards the geometric tail $S\rho^{K+1}$ — systematic energy loss, too-dark interreflections.
  • Russian roulette is unbiased. Continue with probability $q$ and divide the weight by $q$ on survival; reaching-probability $q^k$ and weight $1/q^k$ cancel, so $\mathbb E$ equals the full sum.
  • Finite work. Expected path length $1/(1-q)$ — a randomized stop that turns an infinite sum into a terminating one.
  • Choose $q$ wisely. Variance diverges for $q\le\rho^2$ (fireflies) and cost diverges as $q\to1$; $q=\rho$ (from the throughput) keeps the weight bounded — the practical choice.
  • It’s the renderer’s termination rule — survival = clamped max-channel throughput, with a $1/q$ compensation, skipping the first bounces.
