Blue noise: it's the spectrum, not just the variance

30_blue_noise.cpp — well-spread points lower variance like stratification, but blue noise's real superpower is its frequency spectrum: it pushes error into high frequencies the eye discounts, so the same sample count looks dramatically cleaner.

Independent random samples are white noise: a flat power spectrum, error at every frequency — including the low ones the eye is most sensitive to, which is why naive random sampling looks blotchy and clumpy. Blue noise is a point set whose energy sits in high frequencies and is suppressed at low ones: the points stay evenly separated, no clumps, no holes. That lowers integration variance like stratification — but the deeper win is perceptual: high-frequency error is the kind your visual system averages away.

One-sentence summary. Blue-noise point sets keep samples well-separated (large minimum spacing, no clumps), which both lowers variance on smooth integrands and — crucially — shapes the error spectrum: its structure factor is near zero at low frequencies and rises toward high ones, so the residual noise is the fine-grained kind the eye discounts, the reason renderers use blue-noise sample sets and dither masks for cleaner images at low sample counts.

1. White noise clumps; blue noise spreads

Independent uniform points (white noise, the clumps-and-gaps of example 16) leave some regions crowded and others empty. Blue noise enforces a minimum spacing so the points tile the plane evenly. The C++ file builds it with Mitchell's best-candidate algorithm: to place each new point, propose several random candidates and keep the one farthest from all points placed so far (distances measured toroidally, so the set wraps seamlessly). More existing points ⇒ more candidates, keeping the spacing uniform as the set grows.

2. The spectrum, measured

"Blue" is a statement about frequency. The structure factor $S(\mathbf k)=\frac1N\big|\sum_j e^{-2\pi i\,\mathbf k\cdot\mathbf x_j}\big|^2$ measures how much sampling energy sits at spatial frequency $\mathbf k$. White noise has $S(\mathbf k)\approx1$ at every frequency — a flat spectrum, energy everywhere. Blue noise drives $S$ toward $0$ at low frequencies and lets it rise to $\approx1$ at high ones. The visual system's contrast sensitivity peaks at low-to-mid frequencies, so error parked at high frequencies is far less visible — and tends to average out spatially. That is why blue noise looks cleaner than white noise at the same sample count, even when their total error energy is similar.

3. The variance bonus

Because suppressing low frequencies is the same as removing clumps, blue noise also lowers integration variance on smooth integrands, much like stratification. Integrating $\iint_{[0,1]^2}e^{x+y}=(e-1)^2$ with $N=64$ points:

point set (N = 64)variancevs white
white noise2.32e−21×
blue noise4.78e−34.8×

A $4.8\times$ reduction — useful, but more modest than stratification's or LHS's, and not the headline. Blue noise's distinctive value is the spectrum: two sample sets can have the same variance yet look completely different, and the blue one wins because its error is the kind the eye throws away.

4. In the renderer

Blue noise is everywhere modern renderers fight visible noise. Per-pixel sample seeds are drawn from a blue-noise mask so that neighbouring pixels' errors are decorrelated at low frequencies — the leftover noise looks like fine film grain instead of blotches, and a denoiser handles it far better. The same idea distributes error across a frame (ordered dithering, blue-noise dither masks) and across time in real-time renderers. Best-candidate is the simple offline generator shown here; production uses precomputed blue-noise textures and void-and-cluster masks. The lesson the file drives home is that how the error is distributed in frequency matters as much as how much there is — variance is not the only scoreboard, which is exactly the theme of example 31.

5. The code

The complete sample on GitHub: 30_blue_noise.cpp.

Best-candidate: each new point is the farthest-from-existing of a handful of random proposals.

// place point k as the best of min(k+1, cap) random candidates
for (int i = 0; i < n; ++i) {
    int m = std::min(i + 1, capCandidates);
    P2 best{}; float bestMin = -1.0f;
    for (int c = 0; c < m; ++c) {
        P2 cand{rng.NextFloat(), rng.NextFloat()};
        float md = 1e9f;                                  // nearest existing point
        for (const P2& p : pts) md = std::fmin(md, Dist2(cand, p));   // toroidal
        if (md > bestMin) { bestMin = md; best = cand; }  // keep the most isolated
    }
    pts.push_back(best);
}
Takeaway. Blue noise is a point set with energy concentrated in high frequencies — built here by best-candidate selection, keeping every new point far from the rest. It spreads samples evenly (mean spacing $0.0335$ vs white's $0.0206$), which lowers smooth-integrand variance a modest $4.8\times$, but its real value is the spectrum: low-frequency error is suppressed, so the residual noise is the fine-grained kind the eye discounts. It's why blue-noise masks and dither patterns give cleaner images at the same sample count.
