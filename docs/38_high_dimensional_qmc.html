High dimensions: where QMC (and scrambling) earn their keep

38_high_dimensional_qmc.cpp — path-tracing integrals are high-dimensional (pixel, lens, time, then two numbers per bounce → easily 10–30 dimensions). That is where the sampling strategies diverge most, and where quasi-Monte Carlo hits its limits.

Quasi-Monte Carlo shines in low dimensions, but a path integral has dozens. Two things break there. First, the Halton trap: Halton uses the radical inverse in the $d$-th prime base, and a large base $b$ marches monotonically $0,\tfrac1b,\tfrac2b,\dots$ for its first $b$ samples — so two high dimensions stay almost perfectly correlated, their 2D projection collapsing to a few diagonal stripes. Second, even after you fix that with scrambling, QMC's advantage erodes as the integrand's effective dimension rises — the curse of dimensionality. Both are visible in the numbers, and both point at the same fix that production renderers use: Owen-scrambled Sobol.

One-sentence summary. In high dimensions raw Halton's large prime bases march in lockstep, so a pair of dimensions collapses onto diagonal stripes ($8\times8$ $\chi^2=2102$ vs random's $72$); random digit scrambling — an independent permutation of the base-$b$ digits — breaks the lockstep ($\chi^2=38$) and restores an unbiased randomized-QMC estimator, but the benefit depends on the integrand's effective dimension: on a low-effective-dimension (additive) integrand scrambled Halton wins at every $d$, while on a high-effective-dimension (product) integrand raw Halton detonates ($\sim\!10^6$ error at $d=32$) and every method drifts back toward random's $1/\sqrt N$.

1. The Halton trap in high dimensions

The Halton sequence assigns dimension $d$ the radical inverse $\Phi_{b_d}$ in the $d$-th prime base — bases $2,3,5$ for the first dimensions, but $127,131$ by the thirty-first and thirty-second. 2. Why it collapses to stripes

The radical inverse writes $i$ in base $b$ and reflects its digits about the point: if $i=\sum_k d_k b^k$ then $\Phi_b(i)=\sum_k d_k b^{-(k+1)}$. For every index smaller than the base, $1\le i<b$, there is only the single digit $d_0=i$, so

$$\Phi_b(i)=\frac{i}{b}\qquad(1\le i<b).$$

Now put two high bases together. For $i<127$ the projection onto dimensions 31 and 32 is

$$\big(\Phi_{127}(i),\,\Phi_{131}(i)\big)=\Big(\frac{i}{127},\frac{i}{131}\Big),$$

which is just the point $i\cdot(\tfrac1{127},\tfrac1{131})$ — a set of samples marching in lockstep straight up the line $y=\tfrac{127}{131}x$. Only once $i$ passes $127$ does a second base-127 digit appear and start folding the sequence back across the square; base 131 folds a little later still. With only $N=1024$ points the two bases complete just a handful of these sweeps, so the projection is a few parallel diagonals rather than a filled square — the $\chi^2=2102$ you see in Figure 1. An integrand that depends on dimensions 31 and 32 jointly is then integrated over a 1D smear instead of a 2D area, and the estimate suffers.

3. Scrambling breaks the lockstep

The fix is random digit scrambling: for each dimension and each digit position, draw an independent random permutation $\pi$ of the $b$ possible digit values and apply it, $\Phi_b^\pi(i)=\sum_k \pi_k(d_k)\,b^{-(k+1)}$. The crucial one is the permutation of the leading digit: $\Phi_b(i)=i/b$ becomes $\pi_0(i)/b$, and a genuine permutation (not merely an additive shift) scatters the once-monotonic march across the whole unit interval — the lockstep is gone. Because $\pi$ is uniformly random, the scrambled point is uniform, so the estimator is unbiased and averageable with error bars: randomized QMC, the same principle as the Owen scramble of example 35, applied here to Halton. In the numbers the $8\times8$ $\chi^2$ for dimensions 31 & 32 falls from $2102$ to $37.9$ — from pathological back to indistinguishable-from-random, exactly what you want before asking QMC to do better than random.

4. Effective dimension and the curse

Fixing the projections is necessary but not sufficient — QMC's edge also depends on how the integrand uses its dimensions. An additive function $f=\tfrac1d\sum_j e^{x_j}$ has low effective dimension: it is a sum of 1D pieces, and QMC only needs each 1D projection to be well spread. A product $f=\prod_j|4x_j-2|$ is the opposite — it needs the full joint distribution across all $d$ dimensions at once. Integrating both at $N=4096$ and sweeping the dimension tells the whole story:

Read the two regimes. On the additive integrand scrambled Halton is essentially flat near $8\times10^{-5}$ — its low effective dimension means extra dimensions barely cost anything — while raw Halton climbs from $5.8\times10^{-4}$ at $d=2$ to $4.8\times10^{-3}$ at $d=32$, overtaken by plain random ($1.3\times10^{-3}$) because its high-base projections are the striped mess of Figure 1. On the product integrand the joint distribution is everything: raw Halton's correlated high dimensions produce a wildly wrong estimate ($1.0\times10^{6}$ absolute error at $d=32$), scrambling recovers most of the accuracy ($0.61$), but even it is now only matching random ($0.87$) — no method escapes the curse when the effective dimension is genuinely high. Two lessons fall out: scrambling matters most exactly in high dimensions, and QMC's speed-up is contingent on the integrand's effective dimension, not just its nominal one.

5. The code

The radical inverse whose single leading digit causes the lockstep, and the scrambled version that permutes each digit to break it.

// Raw radical inverse: for i < base there is ONE digit, so Phi(i) = i/base --
// two high bases then march in lockstep -> diagonal stripes.
double RadicalInverse(int base, uint32_t i) {
    double invBase = 1.0/base, invBaseN = 1.0, r = 0.0;
    while (i) { invBaseN *= invBase; r += (i % base) * invBaseN; i /= base; }
    return r;
}

// Scrambled: apply an independent random permutation of the digits at each
// (dimension, position). Permuting the LEADING digit is what breaks the lockstep.
double Scrambler::inv(int d, uint32_t i) const {
    int base = kPrimes[d]; double invBase = 1.0/base, invBaseN = 1.0, r = 0.0;
    for (size_t pos = 0; pos < perm[d].size(); ++pos) {
        int dig = perm[d][pos][i % base];      // <- permuted digit, not i%base
        invBaseN *= invBase; r += dig * invBaseN; i /= base;
    }
    return r;                                  // uniform -> unbiased randomized QMC
}
Takeaway. Path integrals are high-dimensional, and that is where QMC strains. Halton's high dimensions use large prime bases whose radical inverse $\Phi_b(i)=i/b$ marches in lockstep for $i<b$, so a pair of dimensions collapses onto diagonal stripes ($\chi^2=2102$ vs random's $72$). Random digit scrambling permutes each digit — breaking the leading-digit lockstep — and restores an unbiased, near-random projection ($\chi^2=38$). But the payoff depends on effective dimension: scrambled Halton dominates a low-effective-dimension (additive) integrand at every $d$, while on a high-effective-dimension (product) integrand raw Halton detonates to $10^6$ error and every method slides back to random's $1/\sqrt N$. Production path tracers use Owen-scrambled Sobol — a better high-dimensional base than Halton, randomized by exactly this scrambling.
