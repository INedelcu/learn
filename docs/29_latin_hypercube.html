Latin hypercube sampling: stratifying high dimensions

29_latin_hypercube.cpp — full stratification needs Nᵈ cells, hopeless past a few dimensions. Latin hypercube stratifies every axis with just N samples — one per slice per dimension — capturing the 1-D variance that dominates.

Stratification (example 07) crushed 1-D variance by putting one sample per cell. The naive extension to $d$ dimensions — a full grid — needs $N^d$ cells, so even 2 strata per axis is $2^{10}=1024$ samples just to fill a 10-D path-tracing integral once. That is the curse of dimensionality. Latin hypercube sampling sidesteps it: with only $N$ samples it stratifies each dimension independently — every axis gets exactly one sample in each of its $N$ slices — so all the 1-D projections are perfectly stratified for the price of $N$, not $N^d$.

One-sentence summary. Latin hypercube (a.k.a. N-rooks) places sample $i$ at $x_{ij}=(\pi_j[i]+u_{ij})/N$ with an independent random permutation $\pi_j$ per dimension, so each axis is perfectly 1-D-stratified with $N$ samples; for an integrand dominated by its 1-D "main effects" this removes almost all the variance — here a 6-D additive integral drops $3700\times$ — capturing exactly what a full $N^d$ grid would, without the impossible cell count.

1. The curse: a full grid needs Nᵈ cells

A path sample is high-dimensional: pixel $(x,y)$, lens $(u,v)$, a light pick, then two random numbers per bounce — easily 10+ dimensions. Stratifying it as a grid means $k$ strata on each of $d$ axes: $k^d$ cells. Even the coarsest split, $k=2$, explodes as $2^d$. The full-grid cell count races away from any feasible sample budget while LHS holds flat at $N$:

2. Latin hypercube: one sample per slice, per axis

The construction is one line. For sample $i$ and dimension $j$,

$$x_{ij}=\frac{\pi_j[i]+u_{ij}}{N},$$

where $\pi_j$ is an independent random permutation of $\{0,\dots,N-1\}$ and $u_{ij}\sim[0,1)$ jitters within the slice. Per dimension, the $N$ samples land one in each slice $[k/N,(k+1)/N)$ — a perfect 1-D stratification — and the random permutation scrambles which slice pairs with which across axes, so the dimensions stay decorrelated. In 2-D this is the N-rooks pattern: exactly one point in every row and every column. Compare it to plain random, which leaves whole rows and columns empty while doubling up others:

3. Why it works — main effects vs interactions

Write the integrand as a sum of a 1-D part per axis plus the rest: $f(\mathbf x)=\sum_j g_j(x_j)+(\text{interactions})$. The 1-D pieces are the main effects. For sample $i$ the estimator sums over dimensions, and grouping by dimension,

$$\frac1N\sum_i f(\mathbf x_i)=\sum_j\underbrace{\frac1N\sum_i g_j(x_{ij})}_{\text{1-D stratified mean of }g_j}+(\dots),$$

each inner sum is a 1-D stratified estimate of $\int g_j$ — because dimension $j$ has exactly one sample per slice. So LHS reduces every main effect to the stratified variance of example 07 (which falls like $1/N^3$, not $1/N$), and the independent permutations make the per-dimension errors uncorrelated. What LHS cannot remove is the interaction part — variance that depends on two or more axes jointly. The good news: most real integrands are dominated by main effects, so the residual is small.

The C++ test uses a purely additive integrand, $f(\mathbf x)=\sum_{j=1}^{6}e^{x_j}$ over $[0,1]^6$ (exact $6(e-1)=10.30969$) — no interactions at all, so LHS achieves the full 1-D stratification gain on every axis:

A 3739× variance reduction with the same 64 samples — because the additive integrand's variance lives entirely in the 1-D projections LHS stratifies, exactly the per-axis stratification win of example 07, now reaped in six dimensions at once. (Add cross-dimension terms and the gain shrinks toward what those interactions cost; here there are none.)

4. In the renderer

This is why renderers stratify per dimension rather than building an impossible $N^d$ grid. Each random number a path consumes — pixel jitter, lens, light pick, per-bounce direction — is drawn from its own stratified 1-D sequence, so every projection is well covered while the sample count stays at $N$. In practice LHS is usually combined with the low-discrepancy sequences of example 11 (which improve the cross-dimension coverage LHS ignores) and with "padding" — independently stratified or scrambled sequences assigned to different path dimensions. The principle is the same: pay $N$, stratify the dimensions that carry the variance, and accept that the rare high-order interactions go uncaptured.

A fresh permutation per dimension; sample $i$ reads slice $\pi_j[i]$ on each axis.

// one independent permutation of {0..N-1} per dimension (Fisher-Yates)
for (int j = 0; j < d; ++j) {
    std::iota(perms[j].begin(), perms[j].end(), 0);
    for (int k = N-1; k > 0; --k) std::swap(perms[j][k], perms[j][rng.NextInt(k+1)]);
}
// sample i: each axis lands in its own slice perms[j][i], jittered within it
for (int i = 0; i < N; ++i)
    for (int j = 0; j < d; ++j)
        x[j] = (perms[j][i] + rng.NextFloat()) / N;     // perfect 1-D stratification per axis
Takeaway. A full stratified grid costs $N^d$ cells — the curse of dimensionality. Latin hypercube sampling stratifies every axis with just $N$ samples by drawing $x_{ij}=(\pi_j[i]+u_{ij})/N$ with an independent permutation per dimension: one sample per slice per axis (the N-rooks pattern). It reduces each 1-D main effect to the stratified variance of example 07 — a $3700\times$ cut on a 6-D additive integral — while leaving cross-dimension interactions uncaptured. It is the practical reason renderers stratify per dimension instead of gridding.
