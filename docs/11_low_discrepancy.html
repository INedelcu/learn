Low-discrepancy sequences (quasi-Monte Carlo)

11_low_discrepancy.cpp — replace randomness with a deterministic sequence engineered to fill space as evenly as possible at every prefix length, and the error falls nearly as 1/N instead of 1/√N.

Random points clump (example 07): clumps and gaps are wasted information, and they are the $1/\sqrt N$ noise. Stratification fought back by forcing one sample per grid cell. Quasi-Monte Carlo goes further — it drops randomness entirely for a deterministic sequence whose every prefix is already spread out. For smooth integrands the payoff is dramatic: error $\sim1/N$ rather than $1/\sqrt N$, nearly squaring the accuracy for the same sample count.

One-sentence summary. A low-discrepancy sequence (van der Corput in 1-D, Halton in $d$-D) places each new point in the largest remaining gap, so its local density never strays far from uniform (discrepancy $\sim(\log N)^d/N$ instead of random's $1/\sqrt N$); by the Koksma–Hlawka bound, error $\le$ variation$(f)\times$discrepancy, so smooth-integrand error converges nearly as $1/N$.

1. The problem: random points clump

Scatter $N$ independent uniform points and the eye immediately finds dense knots and empty holes. Discrepancy makes that precise: take any sub-box, compare the fraction of points inside it to its area; the discrepancy is the worst such mismatch over all boxes. Random points have discrepancy $\sim1/\sqrt N$. A low-discrepancy sequence keeps it near $1/N$ — every region holds about its fair share of points at every $N$.

2. See it: random vs. Halton

Same number of points, two ways. The random set clumps; the Halton set tiles the square almost evenly. The faint box is a fixed test region of area $0.25$ — watch its “local-density error” (how far the fraction of points inside strays from $0.25$): Halton’s stays tiny, random’s wanders.

3. The van der Corput sequence (1-D)

The 1-D building block is the radical inverse: write the index $i$ in base $b$, then mirror its digits across the radix point. In base 2, $i=6=110_2$ becomes $0.011_2=\tfrac14+\tfrac18=\tfrac38$. The effect is that each new point bisects the largest empty gap:

$$\text{base 2:}\quad 0,\ \tfrac12,\ \tfrac14,\ \tfrac34,\ \tfrac18,\ \tfrac58,\ \tfrac38,\ \tfrac78,\ \dots$$

Step through it and watch the newest point (highlighted) always drop into the widest gap — the opposite of random, which would happily pile two points next to each other.

4. The Halton sequence ($d$-D)

To fill a $d$-dimensional cube, run a van der Corput sequence in each axis but with a different prime base per dimension — 2, 3, 5, 7, … The bases must be coprime, or the dimensions line up into a visible lattice instead of covering the cube. For 2-D that is bases 2 and 3, exactly the right panel of Figure 1:

$$\mathbf x_i=\big(\,\Phi_2(i),\ \Phi_3(i)\,\big),\qquad \Phi_b=\text{radical inverse in base }b.$$

This is the sequence pbrt’s HaltonSampler is built on. It is deterministic — no RNG — yet covers the domain better than any random draw.

5. Why it converges faster

The link between “well spread” and “accurate” is the Koksma–Hlawka inequality:

$$\Big|\,\underbrace{\tfrac1N\textstyle\sum_i f(\mathbf x_i)}_{\text{estimate}}-\underbrace{\textstyle\int f}_{\text{truth}}\,\Big|\ \le\ \underbrace{V(f)}_{\text{variation of }f}\ \times\ \underbrace{D_N^*}_{\text{discrepancy}} .$$

The error is bounded by how much the function varies times how non-uniform the points are. We can’t change $f$, but we can shrink the discrepancy: random’s $D_N^*\sim1/\sqrt N$ gives the familiar $1/\sqrt N$ error, while a low-discrepancy sequence’s $D_N^*\sim(\log N)^d/N$ gives error nearly $1/N$ for smooth $f$ — almost the square of the accuracy at the same $N$.

6. Convergence: 1/√N vs. ~1/N

Integrate $f(x,y)=e^{x+y}$ over the unit square (exact $=(e-1)^2$) and plot the error against $N$ on log–log axes. Random sampling (RMS error over many trials) tracks slope $-\tfrac12$; Halton tracks slope $\approx-1$. Different slope = different convergence order.

7. The catch: high dimensions and randomized QMC

Why renderers don’t use raw Halton everywhere. In high dimensions the small prime bases run out and pairs of dimensions develop visible correlation (the radical inverses in bases 11 and 13 trace slanted patterns before they fill in). Production samplers therefore scramble the sequence — randomized QMC — which restores an unbiased, per-run error estimate while keeping the fast convergence. Owen scrambling (example 35) and high-dimensional QMC pitfalls (example 42) develop this; here we show the raw, unscrambled idea.

8. The code

The complete sample on GitHub: 11_low_discrepancy.cpp.

The whole sampler is the radical inverse; Halton is just two of them with different bases:

// radical inverse of a in base b: mirror a's base-b digits about the radix point
double RadicalInverse(unsigned base, uint64_t a) {
    double invBase = 1.0/base, invBaseN = 1.0, result = 0.0;
    while (a) {
        uint64_t digit = a % base;
        invBaseN *= invBase;
        result   += digit * invBaseN;   // result += digit * b^-(k+1)
        a /= base;
    }
    return result;
}

// Halton point i in 2-D = van der Corput in bases 2 and 3
double x = RadicalInverse(2, i);
double y = RadicalInverse(3, i);

// compare convergence on ∫∫ e^(x+y) = (e-1)^2
//   random:  report RMS error over many trials   → error·sqrt(N) ≈ const   (order 1/√N)
//   Halton:  one deterministic estimate           → error·N      ≈ const   (order 1/N)

The error·sqrt(N) column (random) and the error·N column (Halton) both stay roughly constant — exposing the two different convergence orders directly.

9. In the renderer

Rendering link. This demo renderer (the PathTracingDemo project these docs accompany) keeps sampling simple: it jitters pixel (and lens/light) samples with a Wang-hash RNG — RandomFloat01 (built on WangHash) — i.e. plain random jitter (example 07). Production path tracers and pbrt instead drive those same samples with low-discrepancy sequences (Halton, Sobol) to buy the $\sim1/N$ convergence shown here. Same hook in the pipeline, faster sampler — the upgrade path from this demo to a production renderer.

10. Takeaways

  • Discrepancy is the lever. Error $\le V(f)\times D_N^*$ (Koksma–Hlawka); lowering the discrepancy lowers the error without touching $f$.
  • Radical inverse / van der Corput. Mirror the index’s base-$b$ digits; each new point bisects the largest gap — discrepancy $\sim(\log N)/N$ in 1-D.
  • Halton stacks van der Corput with coprime prime bases per dimension to fill a $d$-cube deterministically.
  • Order, not just constant. QMC error falls $\sim1/N$ vs random’s $1/\sqrt N$ — like stratification (07) it improves the rate, here in any low dimension and from a single deterministic sequence.
  • Scramble for production. Randomized QMC (Owen scrambling, ex. 35) fixes high-dimensional correlation and restores an unbiased error estimate while keeping the speed.
