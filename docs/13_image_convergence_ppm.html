Seeing the noise: image-space convergence

13_image_convergence_ppm.cpp — the capstone. A pixel's color is itself a Monte Carlo integral, so the speckle in a render is exactly the per-pixel estimator variance you've been studying — and it melts as $1/\sqrt N$.

Everything in this series has been about one estimator and its noise. This file makes that noise visible: it renders a tiny scene where each pixel's true color is the average shade over the pixel's little square — an antialiasing integral — and estimates it with $N$ samples per pixel. Pixels deep inside a shape have zero variance (every sample agrees); pixels on an edge disagree, and that disagreement is the speckle. It is the 2-D analogue of a path tracer's noisy penumbrae and glossy highlights.

One-sentence summary. Each pixel is a Monte Carlo integral over its area, so its noise is the estimator variance from example 04 — concentrated wherever the integrand jumps (edges); averaging $N$ samples per pixel makes random noise fall as $1/\sqrt N$, and stratified sampling (example 07) does even better because it spreads samples evenly across each edge.

1. A pixel's color is a Monte Carlo integral

A pixel doesn't cover a single point — it covers a little square of the image. Its correct color is the average of the scene over that square:

$$C_{\text{pixel}}=\frac{1}{\text{area}}\int_{\text{pixel}} \text{shade}(u,v)\,du\,dv\;\approx\;\frac1N\sum_{i=1}^N \text{shade}(u_i,v_i),$$

estimated by sampling $N$ points (subpixel positions) inside the square and averaging — the example-03 estimator, once per pixel. Our scene is deliberately trivial — a few flat-shaded circles over a dark background, “shade” = the top-most circle covering the point — so the only thing creating noise is the geometry of the edges.

2. Where the noise lives

If a pixel lies entirely inside one shape, every subpixel sample returns the same shade: the estimate is exact, variance zero. If a pixel straddles an edge, some samples land on one shape and some on another — they disagree, so the average is uncertain and jitters from run to run. That is the speckle, and it lives only in thin bands along edges.

The edge pixel is a Bernoulli trial. Take a pixel covering fraction $p$ of shade $A$ and $1-p$ of shade $B$. A uniform subpixel sample returns $A$ with probability $p$ and $B$ with probability $1-p$ — a scaled coin flip — so one sample has variance

$$\sigma^2_{\text{pix}}=(A-B)^2\,p(1-p),$$

and the $N$-sample average has variance $\sigma^2_{\text{pix}}/N$, i.e. per-pixel RMS error $|A-B|\sqrt{p(1-p)/N}$. Two things drop out at once: an interior pixel has $p\in\{0,1\}$, so its variance is exactly $0$ (no noise), and the noise peaks at a half-covered edge ($p=\tfrac12$). Averaging over the image, the mean-squared error is $\tfrac1N\,\overline{\sigma^2_{\text{pix}}}$, so the whole-image RMS is $\sqrt{\overline{\sigma^2_{\text{pix}}}}\big/\sqrt N$ — the $1/\sqrt N$ law, its constant fixed entirely by the edges. For this scene only ${\approx}1.6\%$ of pixels have $\sigma^2_{\text{pix}}>0$, and $\sqrt{\overline{\sigma^2_{\text{pix}}}}={0.031}$, matching the flat random RMSE·√N column ($\approx0.030$) in §5.

3. Watch the noise melt

Render the scene live. Pick a sampler and a sample count; the image is computed right here, one Monte Carlo integral per pixel. Crank $N$ up and the edge speckle dissolves. Flip to the error map to see exactly where the noise is — the interiors are black (no error), the edges glow.

4. Random vs. stratified, at equal cost

Stratified jitter places one sample in each cell of a $\sqrt N\times\sqrt N$ grid over the pixel (example 07), so the samples can't clump. At the same $N$ it produces visibly smoother edges than independent random jitter — set the render above to 16 spp and flip between the two samplers, or watch the error map: stratified's edge bands are dimmer.

6. This is the renderer’s progressive averaging

Rendering link. A real path tracer accumulates samples over frames rather than all at once, but it is the identical running mean. In RayGenerator.raytrace the new frame's radiance is blended into the accumulation buffer with lerp(prevRadiance, radiance, 1/(g_ConvergenceStep+1)) — exactly $\bar x_n=\bar x_{n-1}+\tfrac1n(x_n-\bar x_{n-1})$, the incremental average from example 01. Each frame is one more sample per pixel, so the image denoises as $1/\sqrt{\text{frames}}$ — the convergence you just watched, spread over time.

7. The code

The complete sample on GitHub: 13_image_convergence_ppm.cpp.

// each pixel's shade = average over N subpixel samples (its Monte Carlo integral)
double RenderPixel(int px, int py, int N, bool stratified, uint32_t salt) {
    Rng rng; rng.SetSequence((py*W + px)*9781u + salt);   // per-pixel seed
    double sum = 0;
    if (stratified) {                                     // √N × √N jittered grid
        int s = lround(sqrt((double)N));
        for (int j=0;j<s;++j) for (int i=0;i<s;++i) {
            double u=(px+(i+rng.NextFloat())/s)/W, v=(py+(j+rng.NextFloat())/s)/H;
            sum += SceneShade(u,v);
        }
        return sum/(s*s);
    }
    for (int k=0;k<N;++k) {                               // independent random jitter
        double u=(px+rng.NextFloat())/W, v=(py+rng.NextFloat())/H;
        sum += SceneShade(u,v);
    }
    return sum/N;
}
// write each image as a PPM (P6: tiny text header + raw RGB), measure RMSE vs a
// 1024-spp stratified reference, print random/stratified RMSE and RMSE·√N.

It writes reference.ppm plus random_NNNN.ppm / stratified_NNNN.ppm into .\images\ — open random_0001.ppm then random_0256.ppm to watch the speckle melt, or compare random_0016.ppm with stratified_0016.ppm at equal cost.

8. Takeaways

  • A pixel is a Monte Carlo integral over its area; its color is the example-03 estimator, once per pixel.
  • Noise = per-pixel variance, concentrated where the integrand jumps (edges) — zero in flat interiors. The 2-D analogue of noisy penumbrae and glossy highlights.
  • Random noise falls as $1/\sqrt N$ (the ×√N column is flat); 4× the samples ≈ 2× less noise.
  • Stratified does better at equal cost — here ≈$N^{-3/4}$, ~4× lower error at 256 spp — by spreading samples evenly across each edge.
  • It’s the renderer’s progressive averaging: per-frame lerp(prev, new, 1/(step+1)) is the same incremental mean, denoising as $1/\sqrt{\text{frames}}$.
