Sobol sequences & Owen scrambling

35_sobol_owen.cpp — the Sobol sequence is low-discrepancy sampling from a few XORs, and Owen scrambling randomizes it without losing the stratification — giving the modern renderer's sampler: near-1/N convergence, unbiased, with honest error bars.

Halton (example 11) was one low-discrepancy sequence; Sobol is the other, and the one most production renderers use. It is built from binary "generator matrices" so each sample is a few XORs, it stratifies beautifully in 2-D, and it scales to high dimensions better than Halton. Raw Sobol is deterministic, though — no error estimate, and high dimensions eventually show correlation. Owen scrambling fixes both: it randomly permutes the sample digits in a nested way that preserves the stratification, so you keep the fast convergence while making the set a random object you can average and put error bars on.

One-sentence summary. Sobol's first two dimensions form a $(0,2)$-sequence — every dyadic box of area $1/N$ holds exactly one of the first $N$ points, perfect 2-D stratification at each power of two — and Owen scrambling applies a nested random digit-permutation that keeps that stratification while making each point uniform; averaging independent scrambles is randomized QMC: unbiased, with error bars, and on smooth integrands its RMSE falls as $\sim N^{-3/2}$, faster than raw Sobol's $\sim N^{-1}$ and far past random's $N^{-1/2}$.

1. Sobol: stratification from a few XORs

Sobol builds each coordinate by XOR-ing precomputed "direction numbers" selected by the bits of the sample index. Dimension 0 is just the base-2 radical inverse of example 11 — a bit reversal of $i$ — and dimension 1 uses direction numbers $v_k=v_{k-1}\oplus(v_{k-1}\!\gg\!1)$. The result is a $(0,2)$-sequence: every elementary interval — any dyadic box of area $1/N$, be it $1\times N$, $\sqrt N\times\sqrt N$, or $N\times1$ — contains exactly one of the first $N$ points. That is far stronger than a single grid: the set is stratified at every dyadic aspect ratio simultaneously. Below, the same point count placed randomly, by raw Sobol, and by Owen-scrambled Sobol:

2. Owen scrambling: randomize, keep the stratification

Owen scrambling walks the binary tree of each coordinate's digits and flips a random, independent coin at every node, deciding whether to swap the subtree below it. Two consequences fall out of that nested structure:

  • Each point becomes uniform. Randomly permuting the digits of a value sends it to a uniformly distributed point in $[0,1)$, so any single Owen-scrambled estimate is unbiased: $\mathbb E[\hat I]=\int f$. Average $M$ independent scrambles and you have an unbiased estimate with a sample variance — honest error bars, which raw deterministic Sobol cannot give.
  • The stratification survives. A nested digit-permutation maps dyadic boxes to dyadic boxes, so the one-point-per-elementary-interval property is preserved. You keep the QMC convergence — and for smooth integrands Owen's nested scramble actively improves it to $\sim N^{-3/2}$ RMSE.

The file uses the cheap hash-based Owen scramble of Laine–Karras / Burley (2020): a handful of multiply-xor operations on the reversed bits, seeded per dimension. Averaging over seeds is randomized QMC.

Three different convergence orders: fitting slopes gives random $\approx-0.50$, raw Sobol $\approx-1.0$, Owen-Sobol $\approx-1.45$ — close to the theoretical $N^{-1/2}$, $N^{-1}$, and $N^{-3/2}$. Owen scrambling doesn't just randomize Sobol for free error bars; on a smooth integrand it converges meaningfully faster than the raw sequence. At $N=16384$ Owen-Sobol's error is already $\sim$7000× below random's.

4. In the renderer

Owen-scrambled Sobol is the modern default sampler in production renderers (pbrt's ZSobolSampler, and the samplers in Arnold, Cycles, and others). Each pixel draws its path dimensions from the Sobol sequence, Owen-scrambled with a per-pixel seed — so every pixel gets the fast QMC convergence and a decorrelated, randomized set whose remaining error is blue-noise-like (example 30) across the image. The randomization is what makes it unbiased and lets the renderer estimate its own error and drive adaptive sampling (example 17). The PathTracingDemo renderer uses a plain hashed RNG (the random column above); swapping in Owen-Sobol is the single highest-leverage sampler upgrade, turning $N^{-1/2}$ into nearly $N^{-3/2}$ on the smooth parts of the image.

5. The code

The complete sample on GitHub: 35_sobol_owen.cpp.

Sobol is XORs of direction numbers; the Owen scramble is a hash on the reversed bits.

// Sobol dim 1: XOR direction numbers v_k = v_{k-1} ^ (v_{k-1} >> 1)
uint32_t Sobol1(uint32_t i) {
    uint32_t r = 0;
    for (uint32_t v = 1u << 31; i; i >>= 1, v ^= v >> 1)
        if (i & 1u) r ^= v;
    return r;
}
// Hash-based Owen (nested-uniform) scramble — Laine-Karras / Burley 2020
uint32_t OwenScramble(uint32_t x, uint32_t seed) {
    x = ReverseBits(x);
    x ^= x * 0x3d20adeau;  x += seed;  x *= (seed >> 16) | 1u;
    x ^= x * 0x05526c56u;  x ^= x * 0x53a22864u;
    return ReverseBits(x);                       // uniform, and stratification preserved
}
// average over several seeds -> randomized QMC: unbiased, error bars, ~N^-1.5 RMSE.
Takeaway. Sobol generates low-discrepancy points from a few XORs; its first two dimensions form a $(0,2)$-sequence, stratified in every dyadic box. Owen scrambling applies a nested random digit-permutation that makes each point uniform (so estimates are unbiased and averageable) while preserving that stratification (so QMC convergence survives) — and on smooth integrands accelerates the RMSE to $\sim N^{-3/2}$, vs raw Sobol's $N^{-1}$ and random's $N^{-1/2}$. Owen-scrambled Sobol is the modern renderer's default sampler.
