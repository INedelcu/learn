Stratified sampling

07_stratified_sampling.cpp — stop the samples from clumping. Split the domain into equal cells, drop one jittered sample in each, and watch the variance fall far faster than $1/\sqrt N$ — for almost no extra work.

Plain Monte Carlo scatters samples independently, so by sheer chance they bunch up in some places and leave gaps in others. Those clumps and gaps are the variance. Stratification fixes it almost for free: carve the domain into $N$ equal strata and place exactly one jittered sample inside each. The samples can no longer all pile into one region — and for a smooth integrand the error plummets. This is precisely the jitter a path tracer puts on pixel, lens, time, and light samples.

One-sentence summary. Replace $N$ independent draws $x_i=U_i$ with one jittered draw per cell, $x_i=(i+U_i)/N$; the estimator’s variance becomes the sum of the within-stratum variances, and since each stratum is only $1/N$ wide the function barely varies across it — so for smooth integrands the variance falls like $1/N^3$ (error $\propto1/N^{1.5}$) instead of $1/N$, a reduction that grows as $N^2$.

1. The problem: independent samples clump

Drop $N$ independent uniform samples on $[0,1]$ and look at where they land. Some stretches get three or four; others get none. The empty stretches are regions of the integrand you simply didn’t look at this run, and the crowded ones are wasted, redundant effort — together they are the run-to-run noise. Re-roll the draw below and compare: the plain row clumps and gaps differently every time, while the stratified row always keeps one sample per cell, evenly covering the domain.

2. The fix: one jittered sample per stratum

The whole change is in how the sample positions are generated:

$$\text{plain:}\quad x_i=U_i,\qquad\qquad \text{stratified:}\quad x_i=\frac{i+U_i}{N},\quad i=0,\dots,N-1,$$

with $U_i\sim\mathrm{Uniform}[0,1]$ in both. The stratified formula puts cell $i$ at $[i/N,(i+1)/N)$ and jitters the sample uniformly within that cell. The estimate is the same average in both cases, $\hat I=\frac1N\sum_i f(x_i)$, and — crucially — stratification is still unbiased: each cell’s sample is uniform over its cell, so it correctly represents that slice. The jitter (rather than placing the sample at the cell center) is what keeps it unbiased and avoids aliasing.

3. Why it works: variance is the sum of tiny pieces

Because the cells are disjoint and the samples independent, the variance of the stratified estimate is the sum of the within-stratum variances:

$$\operatorname{Var}[\hat I_{\text{strat}}]=\frac1{N^2}\sum_{i=0}^{N-1}\operatorname{Var}\!\big[f(X)\,\big|\,X\in\text{cell }i\big].$$

Here is the leverage. Each stratum spans only $1/N$ of the domain, so a smooth $f$ is nearly linear across it, and the variance of a near-linear function over a cell of width $h=1/N$ is about $(f'h)^2/12$ — it shrinks like $h^2=1/N^2$. Summing $N$ such pieces and dividing by $N^2$:

$$\operatorname{Var}[\hat I_{\text{strat}}]\;\approx\;\frac{1}{12\,N^3}\int_0^1\! \big(f'(x)\big)^2\,dx\;\sim\;\frac{1}{N^3},$$

versus plain Monte Carlo’s $\operatorname{Var}[\hat I_{\text{plain}}]=\operatorname{Var}[f]/N\sim 1/N$. So the standard error falls like $1/N^{1.5}$ instead of $1/\sqrt N$, and the ratio of variances grows like $N^2$ — the more samples, the bigger the win.

Intuition. Plain MC pays for variance that is mostly just the integrand’s overall slope across the whole domain. Stratification removes that big, coarse component for free, leaving only the tiny wiggle inside each cell. The smoother $f$ is, the less is left.

4. The payoff: same cost, far less noise

The file measures it the honest way — run many independent estimates with each method and compare the spread. Below, both methods use $N$ samples per estimate; the histograms are drawn on a shared scale. Both center on the true $I$ (both unbiased), but the stratified distribution collapses to a sliver. Slide $N$ and watch the variance-reduction factor climb roughly as $N^2$.

5. Convergence: faster than 1/√N

Plot the RMS error against $N$ on log–log axes and the two methods separate by their slope: plain MC falls at slope $-\tfrac12$ (the universal $1/\sqrt N$ from example 02), while stratified falls at slope $\approx-\tfrac32$. That steeper line is the headline — stratification doesn’t just shift the constant (as importance sampling did), it improves the rate for smooth 1-D integrands.

6. The catch: the curse of dimensionality

A full stratified grid in $d$ dimensions needs $N^d$ cells to keep $N$ subdivisions per axis. At $d=2$ that is fine (an image plane of pixels), but light paths are high-dimensional — many bounces, each with 2+ random numbers — and $N^d$ explodes. So renderers do not build a full grid. Instead they stratify per dimension (each random dimension gets its own 1-D stratification or a shuffled assignment, as in Latin hypercube sampling, example 29) or switch to low-discrepancy sequences that stay well-spread in high dimensions without a grid (examples 11, 16, 42).

Why not just stratify everything? Because $N^d$ cells means a useful $N$ becomes astronomically expensive past a few dimensions, and most cells would never get a sample. The high-dimensional substitutes keep stratification’s “well-spread” benefit while paying only $O(N)$.

7. This is the renderer’s pixel jitter

Rendering link. The image plane is a 2-D stratification: each pixel is a cell, and the path tracer places one jittered sample inside it per frame rather than sampling pixel positions independently. In RayGenerator.raytrace, GenerateCameraRay offsets the ray within the pixel by a random sub-pixel jitter — exactly the $x=(i+U)/N$ idea in 2-D, with the pixel index playing the role of the cell $i$. The same jitter applies to lens (depth of field), time (motion blur), and light samples: one well-spread sample per sub-cell beats $N$ independent darts, for free.

8. The code

The complete sample on GitHub: 07_stratified_sampling.cpp.

Two estimators per trial, with a Welford pass (example 01) over the trials to measure each method’s variance:

const int N = 64;             // samples per estimate = number of strata
for (int t = 0; t < kTrials; ++t) {
    // plain: N independent uniform draws
    double sp = 0;
    for (int i = 0; i < N; ++i) sp += f(rng.NextFloat());
    double estPlain = sp / N;

    // stratified: one jittered sample inside each of the N cells
    double ss = 0;
    for (int i = 0; i < N; ++i) {
        double x = (i + rng.NextFloat()) / N;   // jittered cell position
        ss += f(x);
    }
    double estStrat = ss / N;

    // Welford over trials → variance of each method's estimates
    /* accumulate estPlain into (meanPlain,m2Plain); estStrat into (meanStrat,m2Strat) */
}
printf("variance reduction factor: %.1fx\n", varPlain / varStrat);

Both means agree with $I\approx1.71828$ (unbiased), but the stratified variance is smaller by a large factor that grows with $N$ — visibly less noise for the same number of samples.

9. Takeaways

  • Clumps and gaps are variance. Independent samples cover the domain unevenly; stratification forces even coverage with one jittered sample per cell.
  • The recipe. $x_i=(i+U_i)/N$ instead of $x_i=U_i$. Still an average, still unbiased — the jitter keeps it so.
  • Variance = sum of within-stratum variances. Each cell is $1/N$ wide, so for smooth $f$ each piece is tiny; total variance $\sim1/N^3$, error $\sim1/N^{1.5}$.
  • It improves the rate, not just the constant. Slope $-\tfrac32$ vs plain’s $-\tfrac12$; the reduction grows as $N^2$. (Contrast importance sampling, which lowered the constant at the same $-\tfrac12$ rate.)
  • Curse of dimensionality. A full grid needs $N^d$ cells; high-dimensional rendering uses per-dimension stratification (Latin hypercube, ex. 29) or low-discrepancy sequences (ex. 11, 16, 42) instead.
  • It’s the renderer’s pixel/lens/time/light jitter — 2-D stratification of the image plane, essentially free anti-noise.
