Importance-sampling a 2D environment map

15_env_map_2d_sampling.cpp — the 2D sequel to example 14. Factor the joint pixel density into a marginal over rows and a conditional within each row, and you can sample any 2D table with two of the 1D binary searches you already have.

An environment map lights a scene: when a ray escapes, it picks up the sky's color in that direction. Most of the sky is dim, but a small bright patch — the sun — carries almost all the energy. Firing shadow and bounce rays uniformly over the sphere wastes nearly all of them on darkness. We want to sample directions in proportion to the map's luminance, so rays land on the sun far more often. Example 14 did this for a 1D table; a map is 2D, and the trick that lifts the 1D machinery to 2D is the marginal–conditional factorization (pbrt's Distribution2D).

One-sentence summary. Write the joint texel density as $p(u,v)=p(v)\,p(u\mid v)$: build one marginal CDF over the row probabilities $p(v)=\sum_u p(u,v)$ and one conditional CDF per row $p(u\mid v)=p(u,v)/p(v)$; to draw a sample, invert the marginal to pick a row, then invert that row's conditional to pick a column — two 1D binary searches (example 14) give one exact 2D sample with density $p(u,v)$.

1. From a 1D table to a 2D map

The map here is a small procedural sky: a dim gradient (brighter near the horizon), one intense sun at texel $(46,9)$, and a fainter glow. It is $64\times32$ texels — the same numbers the C++ file uses. Each texel's luminance is a weight; normalized, they form a 2D probability table $p(u,v)$ with $\sum_{u,v}p(u,v)=1$. The whole job is to draw $(u,v)$ pairs with exactly that probability.

We could flatten the $64\times32$ grid into one length-2048 table and use example 14 verbatim. That works, but the factorization below is cheaper to build incrementally, generalizes cleanly to $N$ dimensions, and is what pbrt and real renderers use — so it is worth seeing directly.

2. The factorization: marginal × conditional

Any joint density factors as a density over $v$ times a density over $u$ given $v$:

$$p(u,v)=p(v)\,p(u\mid v),\qquad p(v)=\sum_{u}p(u,v),\qquad p(u\mid v)=\frac{p(u,v)}{p(v)}.$$

$p(v)$ is the marginal: collapse each row to its total probability — how likely the sample is to land somewhere in row $v$. $p(u\mid v)$ is the conditional: given we are in row $v$, where along it? Each row gets its own conditional distribution. The figure below shows all three at once: the map, the marginal as a column of bars on the right, and the selected row's conditional as bars underneath.

3. Sampling: two inversions, one 2D sample

Sampling is just example 14 done twice. Take two independent uniforms $\xi_1,\xi_2$:

  1. Invert the marginal CDF with $\xi_1$ to get a row $v$ — a binary search over $32$ row probabilities.
  2. Invert row $v$'s conditional CDF with $\xi_2$ to get a column $u$ — a binary search over $64$ column probabilities.

Why does the pair land with the right probability? The row is chosen with probability $p(v)$ and the column, given that row, with $p(u\mid v)$, so by the chain rule the joint probability of $(u,v)$ is

$$p(v)\cdot p(u\mid v)=p(u,v),$$

exactly the table we normalized. The same factor-and-chain idea samples any 3D, 4D, … density: peel off one dimension as a marginal, recurse on the rest as a conditional. Each texel still returns its density $p(u,v)$ (the green readout above), which is what MIS (example 09) needs to weight this strategy against, say, BRDF sampling.

What it costs. One marginal CDF of length $M_H{+}1$, plus $M_H$ conditional CDFs of length $M_W{+}1$ — here $33 + 32\times65$ floats, built once. Each sample is two binary searches: $O(\log M_H + \log M_W)$, branch-coherent, no rejection. That is pbrt's Distribution2D.

5. Why it cuts variance

The point of all this is variance. The file estimates an integral over the map, $I=\sum_{u,v} f(u,v)\,g(u,v)$, with $g$ a smooth factor in $[0,1]$, two ways:

  • Uniform: pick any texel with probability $1/N$, estimator $N\,f\,g$. The map's huge dynamic range (sun $\approx 200$ vs sky $\approx 0.4$) lands inside the estimator — most samples return almost nothing, a rare few return a spike. High variance.
  • Importance: pick a texel with probability $\propto f$, estimator $f\,g/p = (\text{total})\cdot g$. The $f$ cancels: every sample returns the same order of magnitude, scaled only by the smooth $g$. Much lower variance.
The variance, exactly. Both estimators are unbiased ($\mathbb E=I$), and each per-sample variance $\mathbb E[\text{est}^2]-I^2$ has a closed form. Uniform draws texel $(1/N)$ and returns $N f g$; importance draws texel $\propto f$ (so $p=f/\text{total}$) and returns $\text{total}\cdot g$:

$$\operatorname{Var}_{\text{unif}}=N\!\sum_{u,v} f^2 g^2-I^2,\qquad \operatorname{Var}_{\text{imp}}=\text{total}\!\sum_{u,v} f\,g^2-I^2 .$$

Importance trades the spiky $f^2$ for a single power of $f$ — that is the entire win. It is not driven to zero because the smooth $g^2$ survives; only if $g$ were constant would $\sum f g^2$ collapse to $I^2/\text{total}$ and the variance vanish (the perfect-importance limit of example 06). Evaluated on this map the two formulas give $1.38\times10^7$ and $3.98\times10^6$ — the ${\approx}3.5\times$ reduction, recovered from the table without sampling at all.

6. In the renderer

This is how a production renderer lights a scene from an HDR environment. pbrt, for instance, builds exactly this Distribution2D over the map's per-texel luminance, so that whenever a shadow or bounce ray needs a direction toward the sky it is drawn toward the bright parts instead of groping through dark sky — and the returned pdf lets that strategy be combined with BRDF sampling through MIS (example 09): bright small suns want the map's density, glossy surfaces want the BRDF's, and MIS takes the best of both.

Not every renderer bothers. The PathTracingDemo renderer does the simpler thing: its miss shader just looks up the environment cubemap in whatever direction a bounce ray already happens to travel — g_EnvTex.SampleLevel(sampler_g_EnvTex, WorldRayDirection(), 0) — with no luminance-based sampling, so it finds the sun only when a BRDF- or cosine-sampled ray chances to point at it. That is precisely the high-variance baseline this example improves on: building a Distribution2D over the cubemap is the upgrade that would let its rays aim at bright regions on purpose.

That closes the sampling toolkit: inversion and rejection for closed-form densities, 1D tables, and now nD tables by factor-and-chain. Everything a renderer samples — lights, BRDFs, environment maps, participating media — is one of these.

7. The code

The 1D helpers are example 14's, unchanged. The 2D part builds the marginal pV/cdfV and one conditional cdfU[v] per row, then samples row-then-column:

std::vector<float> pV(MH);                          // marginal p(v) = row sums
for (int v = 0; v < MH; ++v) {
    float s = 0.0f;
    for (int u = 0; u < MW; ++u) s += pdf2d[v * MW + u];
    pV[v] = s;
}
std::vector<float> cdfV = BuildCdf(pV);

std::vector<std::vector<float>> cdfU(MH);            // one conditional CDF per row
for (int v = 0; v < MH; ++v) {
    std::vector<float> row(MW);
    for (int u = 0; u < MW; ++u) row[u] = pdf2d[v * MW + u] / pV[v];   // p(u | v)
    cdfU[v] = BuildCdf(row);
}

auto sample2D = [&](int& ui, int& vi) {
    float vc = SampleContinuous(cdfV,     rng.NextFloat()); vi = (int)vc;   // 1) row
    float uc = SampleContinuous(cdfU[vi], rng.NextFloat()); ui = (int)uc;   // 2) column
};
Takeaway. A 2D (or nD) tabulated density is sampled by factoring it into a marginal and conditionals and chaining 1D inversions: pick a row by its total probability, then a column within that row. It is exact, returns its own pdf for MIS, costs two binary searches, and is the environment-map and many-light sampler at the heart of a production path tracer.
