Importance sampling

06_importance_sampling.cpp — the single biggest lever on Monte Carlo noise: spend your samples where the integrand is big by choosing a sampling density shaped like it.

Example 03 showed that the estimator $\frac1N\sum f(X_i)/p(X_i)$ is unbiased for any density $p$. Example 05 showed how to actually draw from a non-uniform $p$. Now we cash that in: by choosing $p$ to resemble $f$, the ratio $f/p$ barely varies from sample to sample, the average stops fluctuating, and the variance — the noise — collapses. This is the same move a path tracer makes when it samples the BRDF lobes instead of firing rays blindly.

One-sentence summary. The estimator averages the ratio $f(X)/p(X)$; if $p$ is shaped like $f$ that ratio is nearly constant so its variance is small, and in the unreachable ideal $p=f/I$ the ratio is exactly $I$ every time (zero variance) — so picking $p$ to track where $f$ is large cuts noise dramatically without introducing any bias.
integrand $f(x)=e^x$ sampling density $p(x)$ ratio $f/p$ (what gets averaged) true value $I=e-1$

1. The lever: any $p$ is unbiased, but not equally noisy

Recall the general estimator and its mean from example 03 — unbiased for any valid $p$:

$$I=\int f(x)\,dx=\mathbb E_{X\sim p}\!\left[\frac{f(X)}{p(X)}\right].$$

Example 04 then told us the noise is set by the variance of the thing being averaged, here the per-sample value $Y=f(X)/p(X)$. So the bias is fixed (zero) no matter what $p$ we pick — but the variance $\operatorname{Var}[Y]$, and therefore the noise, is entirely ours to control through $p$.

2. The idea: flatten the ratio $f/p$

$\operatorname{Var}[Y]$ measures how much $f(X)/p(X)$ jumps around as $X$ varies. Two extremes make it concrete:

  • Uniform $p=1$: the ratio is just $f(x)=e^x$, which sweeps from $1$ to $2.718$ — a lot of variation, so a lot of noise.
  • Ideal $p(x)=f(x)/I$: then $f/p = I$ for every $x$ — a flat line, zero variance. The catch: building this $p$ needs the answer $I$, which is what we’re trying to compute. But it shows the target — make $p$ proportional to $f$ — and any $p$ that roughly tracks $f$’s shape moves you toward it.

Bottom: the ratio $f/p$ that actually gets averaged flattens toward the constant $I$, its variance shrinking. The tick strip shows where samples drawn from that $p$ land — clustering toward the right, where $e^x$ is big.

3. The experiment: uniform vs. a matched line

The file integrates $f(x)=e^x$ on $[0,1]$ (exact $I=e-1\approx1.7182818$) two ways with the same sample budget:

  1. Uniform: $p(x)=1$, so the per-sample value is just $e^X$.
  2. Importance: $p(x)=\tfrac23(1+x)$ — a straight line that leans the same way $e^x$ does. It is easy to sample by inversion (example 05): its CDF is $P(x)=\tfrac{x+x^2/2}{3/2}$, and solving $P(X)=U$ gives
$$X=-1+\sqrt{1+3U}.$$

The per-sample variances come out to roughly

$$\operatorname{Var}_{\text{uniform}}\approx0.242,\qquad \operatorname{Var}_{\text{importance}}\approx0.027,$$

about a 9× reduction from one crude linear density. Since the standard error is $\sqrt{\operatorname{Var}/N}$, that is $\sqrt9\approx3\times$ lower noise for the same $N$ — equivalently, uniform would need ~9× as many samples to match the accuracy.

See it as noise. Below, many independent estimates (each $K$ samples) are histogrammed for both methods. Both bells sit on the true $I$ (still unbiased!), but the importance histogram is far narrower — that width difference is the variance reduction.

4. This is what the path tracer does

The renderer never fires rays uniformly over the hemisphere. A BRDF concentrates outgoing energy in particular directions — a sharp specular lobe, or the cosine-weighted diffuse lobe — and uniform directions would mostly sample directions that contribute almost nothing, leaving the few important ones under-sampled and noisy. Instead it draws directions from a density shaped like the lobe:

Rendering link. In BRDF.hlsl, SampleSpecularGGX draws directions from the GGX/VNDF distribution and SampleDiffuseLambert draws cosine-weighted directions — each is an importance density $p(\omega)$ chosen to match its lobe. The Monte Carlo weight then becomes the flat-ish ratio $f_r\,L_i\cos\theta / p(\omega)$, exactly the $f/p$ of this page. Good lobe sampling is the difference between a clean render and a blizzard of fireflies.

5. The code

The complete sample on GitHub: 06_importance_sampling.cpp.

Two Welford passes (example 01) accumulate the per-sample variance of each method’s estimator value, then compare:

auto f    = [](double x){ return std::exp(x); };
auto pImp = [](double x){ return (2.0/3.0)*(1.0 + x); };           // density ∝ leans like e^x
auto sampleImp = [&]{ return -1.0 + std::sqrt(1.0 + 3.0*rng.NextFloat()); };  // inversion

// Method 1 — uniform:      Y = f(X)/1
for (i ...) { double x = rng.NextFloat();  double est = f(x);          /* Welford on est */ }
double var1 = s1/(kN-1);

// Method 2 — importance:   Y = f(X)/pImp(X),  X ~ pImp
for (i ...) { double x = sampleImp();      double est = f(x)/pImp(x);  /* Welford on est */ }
double var2 = s2/(kN-1);

printf("variance reduction factor: %.2fx\n", var1/var2);              // ~9x

Both m1 and m2 print the same $\approx1.71828$ (unbiased); only the variances differ — var1/var2 ≈ 9.

6. Takeaways

  • Choosing $p$ is the noise knob. Every $p>0$ on $f$’s support is unbiased; the variance — and only the variance — depends on $p$.
  • Match $p$ to $f$. The flatter the ratio $f/p$, the lower the variance. Sampling where the integrand is large is importance sampling.
  • Zero-variance ideal. $p=f/I$ makes $f/p\equiv I$ — unattainable (needs $I$), but it sets the direction: the closer $p\propto f$, the better.
  • Big, cheap win. A crude linear $p$ already gave ~9× variance reduction (~3× less noise) here; a tighter match does even better.
  • It powers the renderer. GGX and cosine lobe sampling are importance sampling of the BRDF; the leftover ratio $f_r L_i\cos\theta/p(\omega)$ is the same $f/p$. When one $p$ can’t cover everything, you combine several — multiple importance sampling (later examples).

References — importance sampling is standard Monte Carlo; see Owen, Monte Carlo theory, methods and examples, ch. 9, and Pharr, Jakob & Humphreys, Physically Based Rendering, §13.10.
