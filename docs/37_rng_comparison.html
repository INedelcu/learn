Comparing random sources: PRNGs vs low-discrepancy

37_rng_comparison.cpp — after Owen-scrambled Sobol (35) and the Wang hash (36), the natural question is "which is the better random number generator?" The honest answer is that they are different categories of thing, and the numbers show exactly why.

A PRNG — the Wang hash, or a PCG — emits white noise: draws that are independent and uniform, with no structure linking one to the next. That independence is what unbiased estimators and honest stochastic decisions (Russian roulette, reflect-vs-refract) require. A low-discrepancy sequence — Owen-scrambled Sobol — is deliberately not independent: consecutive points repel to fill space evenly, which makes integrals converge faster. Owen scrambling adds just enough randomization to keep it unbiased. Comparing them three ways — point distribution, convergence rate, and speed — shows they are not competitors but complements.

One-sentence summary. On an $8\times8$ grid a uniform source scores $\chi^2\approx m-1=63$ (Wang lands at $70$, white-noise clumps), Owen-Sobol scores $0$ (perfectly stratified), and a bad LCG scores $641$ (collapsed onto a lattice) — but the punchline is the rate: two different good PRNGs both converge at $1/\sqrt N$ (their $\text{error}\cdot\sqrt N$ stays flat), while Owen-Sobol runs at $\sim N^{-1.4}$; a better PRNG does not converge faster, better sample placement does, so you use a hashed PRNG for randomness and a low-discrepancy sequence for placement, together.

1. Two different categories of "random"

The confusion in "which RNG is best?" is that the contenders answer different questions. A PRNG is judged by independence: does knowing past draws tell you anything about the next? A low-discrepancy sequence is judged by uniformity: do the first $N$ points cover the domain evenly? These are almost opposite goals. Independent points must occasionally clump — that is what independence looks like — whereas evenly-spread points must be correlated, because "leave a gap here because there is already a point there" is a dependency. The scatter of $1024$ 2D points makes the three categories visible at a glance, and the grid $\chi^2$ statistic turns the picture into a number.

2. Uniformity: reading the $\chi^2$

Drop $N$ points into $m=g^2$ grid cells and compare the counts $O_k$ to the expected $e=N/m$ with Pearson's statistic $\chi^2=\sum_k (O_k-e)^2/e$. For a genuine uniform source each $O_k$ is $\text{Binomial}(N,1/m)$ with variance $e(1-\tfrac1m)$, so

$$\mathbb E[\chi^2]=\sum_{k=1}^{m}\frac{\operatorname{Var}(O_k)}{e}=\frac{m\,e(1-\tfrac1m)}{e}=m-1=63$$

for the $8\times8=64$ cells. Wang's $70.1$ is one draw from that distribution — white noise, exactly as expected. Owen-Sobol reads $0.0$, and that is not luck: its first two dimensions form a $(0,2)$-sequence (example 35), so every dyadic box of area $1/64$ — and the $8\times8$ grid is precisely such a partition, since $8=2^3$ — holds exactly $N/64=16$ points. Zero deviation, $\chi^2=0$ by construction. The bad LCG's $640.8$ is the opposite failure: a too-small multiplier makes consecutive outputs fall on about a dozen lines, so most cells are empty and a few are packed — enormous deviation. Uniformity separates the lattice from the other two, but, crucially, it does not separate Wang from Sobol in a way that predicts integration performance. For that, watch how the error shrinks.

3. The punchline: convergence rate

Integrate the smooth $\int_{[0,1]^2} e^{x+y}\,dx\,dy=(e-1)^2=2.952492$ and measure RMS error over $64$ independent runs. The story is entirely in the exponent. A PRNG obeys the Central Limit Theorem: error $\approx\sigma/\sqrt N$, so $\text{error}\cdot\sqrt N$ should hold constant. Owen-Sobol has no such barrier; on a smooth integrand it reaches $\sim N^{-3/2}$, so $\text{error}\cdot N$ should keep shrinking. Both predictions land:

Meanwhile Owen$\cdot N$ is not flat but falling ($0.22\to0.02$), so Owen-Sobol beats $1/N$: fitting the slope over the whole range gives $\log(3.40\mathrm{e}{-3}/1.24\mathrm{e}{-6})/\log(16384/64)=\log(2749)/\log(256)\approx-1.43$, the $N^{-3/2}$ of Owen scrambling on a smooth function. The lesson is sharp: swapping one good PRNG for another moves nothing; changing the sample placement moves the exponent.

4. Speed, and the synthesis

Cost per sample: Wang $2.34$ ns, PCG $2.28$ ns, Owen-Sobol $10.44$ ns. The scrambled sequence is about $4.5\times$ more expensive per draw — the bit reversals and integer hashes of the scramble are not free. But efficiency (example 31) is variance $\times$ cost, and at $N=16384$ Owen-Sobol's error is $\sim10^{4}$ smaller; to match it a PRNG would need $\sim10^{8}$ times the samples. The extra $4.5\times$ per sample is irrelevant against that. So the framing "PRNG vs Sobol, pick one" is wrong. Use both: a hashed PRNG as the source of independent randomness — the renderer's Wang-seeded per-pixel stream from example 36 — and a low-discrepancy sequence to place those samples in the high-dimensional path space. The PRNG guarantees unbiasedness and clean stochastic decisions; the sequence buys the convergence rate. They do different jobs.

5. The code

The complete sample on GitHub: 37_rng_comparison.cpp.

The two workhorses: the Wang-hash PRNG (white noise) and the Owen scramble that turns a Sobol index into a stratified, unbiased sample.

// PRNG: an integer hash -> white noise. Independent draws, no structure.
uint32_t WangHash(uint32_t s) {
    s = (s ^ 61u) ^ (s >> 16); s *= 9u; s = s ^ (s >> 4);
    s *= 0x27d4eb2du; s = s ^ (s >> 15); return s;
}

// Low-discrepancy: Owen-scramble a Sobol point. A nested digit permutation that
// randomizes (unbiased) while preserving stratification -> ~N^-1.5 convergence.
uint32_t OwenScramble(uint32_t x, uint32_t seed) {
    x = ReverseBits(x);
    x ^= x * 0x3d20adeau; x += seed; x *= (seed >> 16) | 1u;
    x ^= x * 0x05526c56u; x ^= x * 0x53a22864u;
    return ReverseBits(x);
}
// Wang/PCG both converge at 1/sqrt(N); Owen-Sobol at ~N^-1.5. Placement, not
// the generator, sets the rate -- so use a hashed PRNG AND a sequence, together.
Takeaway. A PRNG (Wang, PCG) and a low-discrepancy sequence (Owen-Sobol) are different tools. Uniformity ($8\times8$ $\chi^2$: Wang $70$, Sobol $0$, LCG $641$) tells lattices apart but doesn't rank the two good sources. The rate does: both PRNGs converge at $1/\sqrt N$ (error$\cdot\sqrt N$ flat), Owen-Sobol at $\sim N^{-1.43}$ (error$\cdot N$ still shrinking) — a better PRNG changes nothing, better placement changes the exponent. Owen costs $4.5\times$ more per sample but needs orders of magnitude fewer, so the answer isn't "pick one": use a hashed PRNG for the randomness and a low-discrepancy sequence for the placement.
