Resampled Importance Sampling & reservoirs

18_resampled_importance_sampling.cpp — importance-sample a target you can only evaluate, never invert. Draw cheap candidates, weight them by the target, keep one. This is RIS (Talbot 2005), the engine under ReSTIR.

Importance sampling (example 06) only works if you can sample the density — usually by inverting its CDF (example 14). But the ideal target in rendering, the full integrand $f=\text{BRDF}\cdot L\cdot V\cdot\cos\theta$, has no invertible CDF: you can compute its value at a point, but you cannot draw from it. Resampled Importance Sampling gets you samples approximately distributed by such a target using only cheap proposals plus evaluations — and its streaming form, the weighted reservoir, is the data structure modern real-time renderers (ReSTIR, Bitterli 2020) reuse across pixels and frames.

One-sentence summary. Draw $M$ cheap candidates from a source $q$ you can sample, give each a resampling weight $w_i=\hat p(X_i)/q(X_i)$ measuring how much the evaluable target $\hat p$ wants it, keep one survivor $Y$ with probability $w_j/\sum_k w_k$, and estimate $\hat I=\frac{f(Y)}{\hat p(Y)}\cdot\frac1M\sum_k w_k$ — unbiased for every $M$, equal to plain importance sampling at $M{=}1$, and converging to ideal importance sampling of $\hat p$ as $M$ grows, all without ever inverting $\hat p$.

1. The problem: a target you can't invert

Our toy integrand factors as $f(x)=\hat p(x)\,h(x)$ on $[0,1]$, with

$$\hat p(x)=e^{8x}\quad(\text{sharp — the part worth following}),\qquad h(x)=0.5+x\quad(\text{a smooth leftover}).$$

Here $\hat p$ happens to be invertible, but pretend it isn't (in rendering it never is) — we will only evaluate it. Sampling $x$ uniformly and averaging $f$ is hopeless: $e^{8x}$ swings from $1$ to $e^8\approx 2981$, so almost every uniform sample returns near-nothing and a rare one near $x=1$ returns a spike. That huge dynamic range is exactly what importance sampling by $\hat p$ would cancel — if only we could sample $\hat p$.

2. The RIS recipe

To estimate $I=\int f(x)\,dx$ given an evaluable target $\hat p\ge 0$ and a sampleable source $q$ (here uniform, $q=1$):

  1. Draw $M$ candidates $X_1,\dots,X_M\sim q$.
  2. Weight each by how much the target wants it: $w_i=\hat p(X_i)/q(X_i)$.
  3. Keep one survivor $Y=X_j$ chosen with probability $w_j/\sum_k w_k$.
  4. Estimate $\displaystyle \hat I=\frac{f(Y)}{\hat p(Y)}\cdot\Big(\frac1M\sum_{k=1}^M w_k\Big)$.

The first factor $f(Y)/\hat p(Y)$ is the leftover the target didn't capture — here just $h(Y)$, since $f=\hat p\,h$. The second factor, the average weight, is an unbiased estimate of the target's normalizer $\int\hat p$. Their product is unbiased for every $M$. At $M{=}1$ it reduces to ordinary importance sampling with the source $q$; as $M\to\infty$ the survivor $Y$ becomes distributed ever closer to $\hat p/\!\int\hat p$, so the estimator's variance falls toward that of ideal importance sampling of $\hat p$.

Why it's unbiased — the cancellation. Condition on the $M$ candidates and average over which one survives. Survivor $X_j$ is kept with probability $w_j/\sum_k w_k$, so

$$\mathbb E\big[\hat I \,\big|\, X_{1..M}\big]=\sum_j \frac{w_j}{\sum_k w_k}\cdot\frac{f(X_j)}{\hat p(X_j)}\cdot\frac{1}{M}\sum_k w_k =\frac1M\sum_j w_j\,\frac{f(X_j)}{\hat p(X_j)},$$

and the troublesome $\sum_k w_k$ cancels. Now $w_j=\hat p(X_j)/q(X_j)$, so $w_j\,f(X_j)/\hat p(X_j)=f(X_j)/q(X_j)$ — and each $X_j\sim q$, giving

$$\mathbb E[\hat I]=\frac1M\sum_j \mathbb E\!\left[\frac{f(X_j)}{q(X_j)}\right]=\frac1M\cdot M\!\int f = I,$$

for any $M\ge1$ and any source $q$ whose support covers $f$. At $M{=}1$ it is literally one-sample importance sampling, $f(X_1)/q(X_1)$. Notice $\hat p$ appears only inside the weights — it is never inverted, only evaluated.

3. The survivors become distributed by the target

That convergence is the whole idea, and you can watch it. Each RIS draw produces one survivor $Y$; histogram many of them. At $M{=}1$ the survivor is just the lone uniform candidate, so the histogram is flat. Raise $M$ and the survivors pile up under $\hat p$ — the resampling has reshaped a uniform source into (approximately) the target, using only evaluations of $\hat p$.

4. Variance: marching toward ideal importance sampling

Because the survivor approaches $\hat p$, the estimator's variance approaches the variance of ideal IS — sampling $\hat p$ directly (its only remaining noise is the smooth $h$). Here $\hat p$ is an exponential, so we can invert it to measure that ceiling. The C++ run, estimating $I=512.31$ with a million trials:

5. The streaming form: a weighted reservoir

Step 3 — "keep one survivor with probability $w_j/\sum_k w_k$" — looks like it needs all $M$ weights at once. It doesn't. Weighted reservoir sampling does the identical selection in one pass with $O(1)$ memory: hold a running survivor $y$ and weight-sum $W$; for each new candidate, add its weight and replace the survivor with probability $w/W$. After processing the stream, $y$ is the RIS survivor — the candidates were never stored.

The streaming and array forms give the same distribution and the same variance (the C++ file confirms the reservoir at $M{=}16$ matches array-RIS at $M{=}16$: variance $5.74\text{e}4$ vs $5.75\text{e}4$). The $O(1)$ memory is what makes the next step possible.

6. ReSTIR: reuse the reservoirs

A reservoir is a tiny, self-contained summary of "the best light sample I found, and how much total weight I saw." Because it is $O(1)$, you can merge two reservoirs — feeding one's survivor into the other as a single weighted candidate is itself a RIS step. ReSTIR (Bitterli et al. 2020) exploits this relentlessly: each pixel keeps a reservoir, then merges it with its neighbors' reservoirs (spatial reuse) and with its own reservoir from the previous frame (temporal reuse). After a couple of merge passes a pixel has effectively sampled hundreds of lights while tracing only a handful of rays itself.

A classic path tracer like PathTracingDemo samples each light or BRDF lobe independently per pixel and never reuses that work — exactly the $M{=}1$, no-reuse end of this spectrum. RIS and reservoir reuse are the upgrade that turns many-light direct lighting from prohibitive into real-time; they are an advanced topic, but every piece is something this series has already built.

7. The code

The array RIS and the streaming reservoir are the same arithmetic; the reservoir just keeps the running survivor instead of an array.

// --- array RIS for one estimate, M candidates (reservoir-style pick) ---
double wsum = 0.0, chosen = 0.0;
for (int i = 0; i < M; ++i) {
    double x = rng.NextFloat();              // X ~ q (uniform, q = 1)
    double w = pHat(x);                      // w_i = p_hat(x)/q(x)
    wsum += w;
    if (rng.NextFloat() < w / wsum) chosen = x;   // keep survivor w.p. w/wsum
}
double est = h(chosen) * (wsum / M);         // (f(Y)/p_hat(Y)) * mean(w)

// --- the same thing as an O(1) streaming reservoir (what ReSTIR stores) ---
struct Reservoir {
    double y = 0.0, wsum = 0.0; int M = 0;
    void update(double x, double w, Rng& rng) {
        wsum += w; ++M;
        if (rng.NextFloat() < w / wsum) y = x;     // replace survivor w.p. w/wsum
    }
    double estimate() const { return h(y) * (wsum / M); }
};
Takeaway. RIS importance-samples a target you can only evaluate: draw $M$ cheap candidates, weight them by the target, keep one. It is unbiased at every $M$ and converges to ideal importance sampling as $M$ grows. Its streaming reservoir form needs $O(1)$ memory, so reservoirs can be merged and reused across pixels and frames — that reuse, ReSTIR, is the state of the art for real-time many-light rendering, and it is built entirely from the importance-sampling ideas in this series.
