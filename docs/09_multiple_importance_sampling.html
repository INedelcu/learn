Multiple Importance Sampling (MIS)

09_multiple_importance_sampling.cpp — when no single density fits the whole integrand, use several and combine them so the result inherits the best of each — and provably never blows up.

Importance sampling (example 06) shines when one density matches the integrand. But real integrands have several features at once, and a density that nails one feature can be catastrophic on another. The textbook rendering case: shading a glossy surface, you can sample the BRDF (great for sharp reflections, terrible for small bright lights) or sample the light (great for small lights, terrible for near-mirror reflections). Each strategy explodes exactly where the other excels. MIS (Veach & Guibas, 1995) lets you use both and weight them so neither spike survives.

One-sentence summary. Take one sample from each strategy and combine them with weights that sum to $1$ everywhere (so it stays unbiased); the balance heuristic $w_i=p_i/\sum_j p_j$ makes each term collapse to $f/\sum_j p_j$ — a bounded weighted average — so wherever a density is tiny (and its $f/p$ would explode) its weight is equally tiny and muzzles the spike.

1. The problem: each strategy fails where the other wins

One density can importance-sample one feature. Put two features in the integrand and any single density is starved on at least one of them — and a starved density is exactly the recipe for a variance explosion: it almost never samples that region, but when it does, $f/p$ is enormous because $p$ there is tiny.

2. The combined, unbiased estimator

Take one sample from each strategy per iteration and add them up, each scaled by a weight:

$$F=w_1(X_1)\,\frac{f(X_1)}{p_1(X_1)}+w_2(X_2)\,\frac{f(X_2)}{p_2(X_2)},\qquad X_1\sim p_1,\ X_2\sim p_2.$$

As long as the weights sum to one at every point, $w_1(x)+w_2(x)=1$, the estimator is unbiased — $\mathbb E[F]=\int f$. (Each term contributes $\int w_i f$, and summing over $i$ rebuilds $\int f$.) So we are free to choose any partition-of-unity weights; the only question is which choice minimizes variance.

3. The balance heuristic, and why it tames spikes

Veach & Guibas’s balance heuristic weights each strategy by how likely it was to have produced that point:

$$w_i(x)=\frac{p_i(x)}{p_1(x)+p_2(x)} .$$

Substitute it back and watch the troublesome $1/p_i$ cancel — each term becomes the same bounded quantity:

$$w_i(x)\,\frac{f(x)}{p_i(x)}=\frac{p_i(x)}{p_1(x)+p_2(x)}\cdot\frac{f(x)}{p_i(x)}=\frac{f(x)}{p_1(x)+p_2(x)} .$$
The spike is muzzled. The danger in single-strategy sampling is a region where $p_i$ is tiny, so $f/p_i$ explodes. Under the balance heuristic that same term is $f/(p_1+p_2)$ — and if $p_1$ is tiny there, $p_2$ is the one that’s large, so the denominator stays healthy and the value stays bounded. The weight is small exactly where the naive estimator would blow up. No single sample can ever contribute a giant value.

4. A self-contained experiment: two peaks

Everything here has a closed form so we can check it. Build two sharp peaks from truncated exponentials on $[0,1]$ (sharpness $a$, normalizer $Z=1-e^{-a}$):

$$p_1(x)=\frac{a\,e^{-a x}}{Z}\ \text{(peak at 0)},\qquad p_2(x)=\frac{a\,e^{-a(1-x)}}{Z}\ \text{(peak at 1)},$$

and make the target a weighted sum of them, so the integral is exact:

$$f(x)=A\,p_1(x)+B\,p_2(x)\quad\Longrightarrow\quad \int_0^1 f=A+B .$$

With $A=1,B=4$ the answer is $5$. $p_1$ samples the left peak beautifully but is starved at the right peak (and vice versa), so each strategy alone is noisy — and MIS should fix it. Each method gets the same budget of 2 samples per estimate.

Bottom: the value each estimator assigns at $x$ — the single-strategy ratios $f/p_1$ and $f/p_2$ rocket off-scale where their density starves, while the MIS term $f/(p_1+p_2)$ stays calmly between $A$ and $B$ across the whole domain. Slide the sharpness to make the peaks (and the single-strategy spikes) more extreme.

6. The payoff — and a road to zero variance

Run many 2-sample estimates with each method. All three are unbiased (centered on $I=A+B$), but the single strategies have heavy, noisy tails from their rare giant samples, while MIS clusters tightly. Slide the peak imbalance: as $A\to B$ the MIS term $f/(p_1+p_2)$ becomes the constant $A$, so its variance drops all the way to zero — the same ideal limit as importance sampling, now reached by combination.

7. The renderer’s cousins of this idea

Rendering link. The path tracer applies the same “combine strategies, weight by how likely each was” logic. In Shading.hlsl, each opaque hit selects a lobe with probability proportional to its luminance (clamped to $[0.1,0.9]$) and divides the weight by that selection probability — discrete one-sample strategy selection, where the selection pdf plays the MIS-weight role. Separately it samples a light directly (next-event estimation), the “sample the light” strategy from Figure 1. Full BRDF-vs-light MIS weighting is the natural next step on exactly these two strategies.

8. The code

The complete sample on GitHub: 09_multiple_importance_sampling.cpp.

// densities, target, and samplers (truncated exponentials; p2 is the mirror of p1)
double Z  = 1 - exp(-a);
double p1(x){ return a*exp(-a*x)/Z; }            // peak at 0
double p2(x){ return a*exp(-a*(1-x))/Z; }        // peak at 1
double f (x){ return A*p1(x) + B*p2(x); }        // ∫ f = A + B
double sampleP1(){ return -log(1 - U()*Z)/a; }   // invert the CDF
double sampleP2(){ return 1 - sampleP1(); }      // mirror

// (1) p1 only:  2 samples, average f/p1   (3) MIS: 1 from each, balance heuristic
estOnly1 = 0.5*( f(xa)/p1(xa) + f(xb)/p1(xb) );
double x1 = sampleP1(), x2 = sampleP2();
double w1 = p1(x1)/(p1(x1)+p2(x1));
double w2 = p2(x2)/(p1(x2)+p2(x2));
estMIS = w1*f(x1)/p1(x1) + w2*f(x2)/p2(x2);      // = f(x1)/(p1+p2) + f(x2)/(p1+p2)

Output (5M trials): all three means hit $5$, but MIS variance (≈ $0.24$) is about 2000× below the best single strategy — $p_2$-only ≈ $480$, $p_1$-only ≈ $8000$ — and with $A=B$ it is exactly zero.

9. Takeaways

  • One density isn’t enough for multi-feature integrands; each importance strategy explodes where it starves.
  • MIS combines strategies with partition-of-unity weights ($\sum_i w_i=1$), keeping the estimator unbiased.
  • The balance heuristic $w_i=p_i/\sum_j p_j$ collapses each term to the bounded $f/\sum_j p_j$ — small weight exactly where $f/p_i$ would blow up.
  • Robust by design. MIS provably never does much worse than the best single strategy, and is typically far better; here ~2000× lower variance than the best single strategy, and exactly zero when the peaks are balanced.
  • It’s the BRDF-vs-light combination of real path tracers, and a cousin of the renderer’s per-hit luminance-weighted lobe selection.
